# Overriding iGEM CSS

For Overriding iGEM CSS, iGEM provides you with some code already in the Default Wiki Settings in your home template: `http://[YEAR].igem.org/Template:[TEAM]` page. 

```
#home_logo, #sideMenu { display:none; }
#sideMenu, #top_title, .patrollink  {display:none;}
#content { margin-left:0px; margin-top:-7px; padding:0px; width:100%;}
body {background-color:white; }
#bodyContent h1, #bodyContent h2, #bodyContent h3, #bodyContent h4, #bodyContent h5 { margin-bottom: 0px; }
```

This along with another set of overrides that I have compiled from my time modifying the wiki:

With that, your page is now fully customizable the way you want it to be. 

In short, we put the above two code snippets into a template page and put that in the page we wanted to change (EP1): 

```
{{Washington/Test}}

<html>
    <link href="http://2018.igem.org/Template:Washington/Testtest?action=raw&ctype=text/css" type="text/css" rel="stylesheet" />
    <h2 id="uw-subheader">This is the example page one</h2>
    <p id="uw-content">Content content content content content</p>
</html>
```

That's it!

## Using the inspector to debug styles

Sometimes there still might be lingering iGEM CSS that wasn't fixed by the script I provided. If you ever encounter this, fear not! Use rightclick-inspect and a menu on the right or bottom will appear. I use Chrome, but it might be similar in Firefox or other modern day browsers.

In the debugger, there will usually be a button that lets you select an element on the page to inspect it more. On Chrome, it is on the top left of the debugger, and you can also access this instantly using Shift-Control-C. Now you can hover over the thing that is giving you problems. 

In this case, my header 2 has some space above it I want to get rid of! If I use the shift-ctrl-c, and select it, I can go over to the "Computed" tab on the side, and see a box model of it. There, I can see that there is padding: 15 on it! I can override this in my CSS, by adding `padding: 0` to my `#uw-subheader`. 

<Img src={require('./imgs/debugger.png')} alt={'debugger'}/>

Using the debugger, you can see what styles are going on, and trace back to the origin of it. You can also just override the styles by setting it on that object. CSS will override existing CSS on the element and save the most recent change.
